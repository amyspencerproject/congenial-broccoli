Connect to SQL Server 2019

If you plan on using the SSH tunneling options, first establish a connection to your SQL Server without any of the optional SSH tunneling credentials. After establishing a connection you can set up SSH tunneling.

Create a new database called precog where the data can be written. The user associated with this database needs to have read and write permissions and be able to create schemas, drop and truncate tables, and append and insert records. Make sure your firewall accepts a connection from 44.205.118.35/32. The “/32” restricts the accepted IP to only an exact match.
